# v0.7.1 — mermaid natural-width render (no more cramped text)

Wide mermaid diagrams now render at natural width and scroll horizontally inside the preview pane, matching how GitHub renders them. Pre-v0.7.1, large flowcharts had their text labels squashed because mermaid's default `useMaxWidth: true` set the SVG to `width:100%` with a viewBox, and our `.mermaid-block svg { max-width: 100% }` CSS reinforced that — the result was uniform shrink-to-fit that ate label legibility on anything bigger than the pane.

CSS-only + init-config fix. No DOM contract change. The existing morphdom mermaid-block cache (`data-mermaid-src` keyed re-use) still skips re-render on unchanged blocks.

## Highlights

| Area | Change |
|---|---|
| `src/lib/mermaid.ts` | `mermaid.initialize({ … })` now sets `useMaxWidth: false` per diagram type for all 17 v10 diagram families: `flowchart`, `sequence`, `gantt`, `journey`, `class`, `state`, `er`, `pie`, `requirement`, `c4`, `mindmap`, `timeline`, `gitGraph`, `sankey`, `xyChart`, `quadrantChart`, `block`. Unknown keys are ignored by mermaid, so the config is forward-safe. |
| `src/app.css` | `.mermaid-block svg { max-width: 100%; height: auto; }` → `.mermaid-block svg { display: block; margin: 0 auto; }`. Dropping `max-width` lets the natural SVG size win; the parent `.mermaid-block { overflow-x: auto; text-align: center }` (unchanged) scrolls when wider than pane and centers when narrower. |

## Why this exists

User pasted a wide flowchart from another mermaid renderer (`mermaid.live` and GitHub's own preview both showed it legibly). The same diagram in gh-md-editor came back as a postage stamp of unreadable text, even though the diagram itself parsed and rendered. Root cause was the double-clamp: mermaid's `useMaxWidth: true` default + our own `max-width: 100%` CSS rule, both forcing the diagram to fit pane width via uniform scaling rather than allowing the container to scroll.

Option B (pan/zoom via `panzoom` lib) was discussed and deferred. Natural-width + horizontal scroll is the GitHub-compatible baseline; pan/zoom can layer on later if wanted.

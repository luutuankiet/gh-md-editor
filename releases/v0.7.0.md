# v0.7.0 — per-pane dark mode + editor readability fix

Each of the three panes (editor / preview / outline) now has its own dark-mode toggle in the top-right corner — light / dark / auto, persisted to `localStorage`. The editor pane finally gets a real dark theme too: pre-v0.7 it rendered light-mode hex syntax colors (#cf222e, #0550ae …) against whatever surface the browser auto-picked, so when the OS sat in dark mode you got near-invisible tokens against a dark background. v0.7 bundles a GitHub-dark palette (#ff7b72, #79c0ff, #d2a8ff, #ffa657…) plus a dark editor surface, hot-swapped through a CodeMirror `Compartment` on toggle.

Cycle order is light → dark → auto, default `auto` so existing users see no visible change until they click. Storage keys: `gmd:theme:editor`, `gmd:theme:preview`, `gmd:theme:outline`.

## Highlights

| Area | Change |
|---|---|
| Editor (CodeMirror) | New `editorThemeExtensions(theme)` in `src/lib/editor-theme.ts` returns the bundle (HighlightStyle + fallback + EditorView.theme chrome) per side. Compartment in `Editor.svelte` swaps on prop change — no state rebuild. Dark variant gives back: heading hierarchy (red/blue/purple/orange), strings (#a5d6ff), numbers (#79c0ff), comments (#8b949e italic). |
| Preview (markdown-body) | Both `github-markdown-light.css` and `github-markdown-dark.css` pulled inline via Vite's `?inline`, then scoped at runtime: every `.markdown-body` selector is rewritten to `.markdown-body.theme-{light,dark}` before being injected as a `<style>` block. The auto `github-markdown.css` import is gone — it used `@media (prefers-color-scheme: dark)` and couldn't be overridden per-pane. |
| Outline | Same class-based pattern — `@media (prefers-color-scheme: dark)` blocks replaced with `.outline-pane.theme-dark` selectors. Toggle docks in the existing outline header alongside `?` / `−` / `+`. |
| GitHub-flavored alerts | Switched from `github-colors-dark-media.css` to `github-colors-dark-class.css`, then re-bind the same `--color-note` / `--color-warning` / … vars under `.preview-container.theme-dark` so NOTE/TIP/WARNING blocks follow the preview pane's toggle instead of OS pref. |
| App.svelte | Owns three `ThemeChoice` $state vars + a live `systemDark` MediaQueryList listener so 'auto' panes flip when the OS does. Derives effective theme and passes down + a cycle callback per pane. |

## Why this exists

The screenshot that triggered v0.7.0 showed dark OS + light editor highlight → syntax tokens disappearing into the surface. The fix had to be per-pane rather than global because the project's whole shape is independent panes — forcing the whole app dark/light by OS pref defeats that.

## Storage

Missing keys default to `'auto'`. No migration needed; pre-v0.7 users land on the same OS-following behaviour they had before, then opt in by clicking a toggle.

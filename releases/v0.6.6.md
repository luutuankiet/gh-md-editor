# v0.6.6 — Editor-pane font-size hotkey (Cmd+= / Cmd+- / Cmd+0)

**The ask.** Per-pane font size control. Bump editor text up or down while editing without dragging the preview pane along — heading hierarchy in the rendered side stays readable, while the editor text scales to match whatever the eyes need that hour. VS Code parity: Cmd+= zooms in, Cmd+- zooms out, Cmd+0 resets.

**The constraint.** Browser zoom (Cmd+= / Cmd+-) is the elephant in the keymap. Without preventDefault firing at the DOM level, the whole page zooms instead of just the editor. CodeMirror's `keymap.of([...])` entries with `preventDefault: true` work in theory, but the DOM keydown intercept fires earlier and is more reliable — same pattern that fixed the Mac Opt+Z `Ω` bug in v0.5.0.

## Implementation

CodeMirror 6 `Compartment` lets the font-size theme reconfigure live without rebuilding the EditorState:

```typescript
const fontSizeCompartment = new Compartment();

function makeFontSizeTheme(px: number) {
  return EditorView.theme({
    '.cm-content': { fontSize: px + 'px' },
    '.cm-gutters': { fontSize: px + 'px' },
  });
}

function applyFontSize(vw: EditorView, px: number) {
  const clamped = Math.max(9, Math.min(32, px));
  localStorage.setItem('gmd:editor-font-size', String(clamped));
  vw.dispatch({
    effects: fontSizeCompartment.reconfigure(makeFontSizeTheme(clamped)),
  });
}
```

Dispatched via the existing DOM keydown handler that already owned the Mac Opt+Z workaround:

```typescript
keydown: (event, vw) => {
  // ... Alt+Z toggle wrap (v0.5.0) ...
  const mod = event.metaKey || event.ctrlKey;
  if (mod && !event.altKey) {
    if (event.code === 'Equal') { event.preventDefault(); bumpFontSize(vw, +1); return true; }
    if (event.code === 'Minus') { event.preventDefault(); bumpFontSize(vw, -1); return true; }
    if (event.code === 'Digit0') { event.preventDefault(); resetFontSize(vw); return true; }
  }
  return false;
}
```

`event.code` is layout-independent (`Equal` / `Minus` / `Digit0`), so Cmd+= works the same on QWERTY, AZERTY, Dvorak, etc. — no key-character normalization gotchas like the Mac Opt+Z → `Ω` issue.

## Why preview is excluded

The request was explicit: scale editor text only, leave preview alone. Pragmatic reason: the preview's heading-hierarchy contrast (h1 18em / h2 11em / body 13em via `github-markdown-css`) only reads as hierarchy when the spacing between sizes is preserved. Uniform-zooming the preview compresses that contrast in absolute pixels — a 32px h1 next to 24px body text looks much flatter than a 26px h1 next to 13px body text. Editor-only zoom sidesteps the issue entirely.

Implementation-wise this comes for free: the Compartment lives inside `Editor.svelte`, so it touches `.cm-content` and `.cm-gutters` selectors — nothing in `Preview.svelte` is reachable.

## Persistence

Follows the same pattern as the existing word-wrap toggle: localStorage key `gmd:editor-font-size`, written on every change, read on mount, gracefully degraded on quota/disabled-storage.

## Files changed

- `src/components/Editor.svelte` — Compartment + state + theme factory + apply/bump/reset helpers; DOM keydown intercept extended; keymap.of([…]) extended with `Mod-=` / `Mod-Equal` / `Mod--` / `Mod-Minus` / `Mod-0`; base theme `fontSize: '13px'` removed (Compartment owns it now).
- `src/components/ShortcutsDialog.svelte` — two new rows with VS Code parity badge: `Cmd/Ctrl + = / -` zoom, `Cmd/Ctrl + 0` reset.
- `package.json`, `public/userscript-installer.html` — version + footer bump 0.6.5 → 0.6.6.

## What this release is NOT

Userscript unchanged. Preview pane unchanged. Per-pane preview zoom is intentionally NOT in this release — see "Why preview is excluded" above. If a future need arises (e.g. someone wants preview-only zoom while keeping editor at 13px), it's a separate Compartment in `Preview.svelte` with its own keymap — straightforward addition, just not in scope here.

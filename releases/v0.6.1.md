# v0.6.1 ‚Äî non-destructive fallback + GitHub-alert inline-body normalization

Two hotfixes surfaced after rendering a real org PR through v0.6.0:

## Bug 1 ‚Äî `> [!NOTE] body` on the same line rendered broken

GitHub's native renderer is lenient: writing `> [!NOTE] **Headline:** body` with the body inline on the same line as the marker renders correctly on github.com. `markdown-it-github-alerts` v1.0.1 still recognizes the marker, but the resulting alert layout breaks ‚Äî title-styling bleeds into the body, body indentation goes off, etc.

Fix: a tiny pre-processor in `src/lib/markdown.ts` splits `> [!TYPE] body` into the canonical two-line form `> [!TYPE]\n> body` before passing source to markdown-it. Covers all five GitHub alert types (NOTE / TIP / IMPORTANT / WARNING / CAUTION), case-insensitive.

The source-line offset introduced by inserting a synthesized body line is tracked in an `insertions` array and threaded through markdown-it's env. The `source_line_map` core rule subtracts insertions at-or-before each token's line so editor‚Üîpreview navigation (click-to-reveal, outline jumps, scroll-sync) keeps pointing at the user's original line numbers.

## Bug 2 ‚Äî fallback link destructively replaced the `<img>` element

v0.6.0's `replaceWithFallback()` did `img.replaceWith(link)`, which dropped the original `<img src="https://github.com/user-attachments/...">` element from the DOM tree. Two consequences:

- Copying/exporting preview HTML out of the editor lost the canonical image markup; the user only got the fallback text.
- Image attributes (`width`, `height`, `alt`) attached to the original element disappeared.

Fix: rename `replaceWithFallback` ‚Üí `applyFallback`. The new function wraps the `<img>` in `<span class="gh-asset-shell gh-asset-shell-fallback">` and appends a sibling `<a class="gh-asset-fallback">` link. CSS in `Preview.svelte` hides the img (`display: none`) and shows the link as the user-facing affordance. The original `<img>` stays in the DOM tree, so:

- Copy/export of preview HTML preserves the canonical `<img src="...">` markup.
- Image attrs stay attached to the original element.
- A future re-render once the userscript installs can recover and resolve the image without losing the source-of-truth element.

Idempotent: re-runs of the walker skip already-wrapped images.

- `src/lib/gh-asset-resolver.ts` ‚Äî `applyFallback()` replaces `replaceWithFallback()`
- `src/components/Preview.svelte` ‚Äî CSS for `.gh-asset-shell` / `.gh-asset-shell-fallback > img`

## v0.7.7 — line-layout + GFM soft breaks for agentic-prompt markdown

- Arbitrary XML-like tags (`<active_task>`, `<vision>`, `<some name of tag>`, etc.) now render as literal text in their own visual paragraph at the editor's source line, with right-click reveal-counterpart working on them. Standard HTML tags (`<details>`/`<kbd>`/etc.) pass through unchanged.
- GFM-style soft breaks: every `\n` in source emits a `<br>` in preview, so editor line layout matches preview line layout (matches how GitHub renders issue/PR comments).
